① SSR & Hydration

Turn a client-only Angular app into a server-rendered one. The server sends complete HTML (great for SEO + first paint); the browser then hydrates it instead of re-rendering.

② SEO meta tags

One small service writes every tag a page needs. Because it runs during SSR, crawlers receive a fully-populated <head> without executing JavaScript.

⚠️
OG image gotcha: social crawlers (LinkedIn, Facebook, X) generally do not render .svg for og:image. Use a raster PNG/JPG around 1200×630 with an absolute URL. This demo ships SVG placeholders for simplicity — swap them for real images before sharing links publicly.

③ Canonical URL & JSON-LD

Canonical tags fix duplicate-URL problems; JSON-LD unlocks rich results (stars, price, FAQ).

④ TransferState (skip the double fetch)

Without this, SSR fetches data on the server and the browser fetches it again after hydration. TransferState ships the server result inside the HTML so the client reads it for free. Angular's built-in HttpClient transfer cache already handles many GET requests; manual TransferState is still useful for custom data sources or explicit cache control.
