Why duplicate requests happen

During SSR, Angular runs your components on Node and HttpClient fetches JSON. After hydration, the browser bootstraps the same app and subscriptions run again — another fetch unless you reuse server data via TransferState.

In modern Angular, provideClientHydration() already includes an automatic HttpClient transfer cache for normal GET/HEAD requests. This lab disables that automatic cache only for products.json so the manual TransferState pattern is visible.
