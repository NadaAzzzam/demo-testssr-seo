Which tag controls which part of the result?

Result partHTML tagIf missing…
Blue clickable headline <title> / og:title Google guesses from <h1> or domain
Grey description snippet meta name="description" Google scrapes random page text
Indexing / snippet rules meta name="robots" Private pages may be indexed, or previews may be limited unexpectedly
Green URL / breadcrumb link rel="canonical" Duplicate-content risk across URLs

Social share card og:* / twitter:* Ugly bare-link preview on LinkedIn/Slack
💡
The SSR connection: crawlers receive HTML first, and not every crawler reliably renders slow client JavaScript before indexing. With SSR ON, the title, description and content are already in the HTML — so the snippet above is reliable. With SSR OFF, the crawler may only see <app-root></app-root> and your result looks empty. Try the toggles above and compare the “What Googlebot downloads” box.
Part 2

Getting your domain into Google Search Console

Before Google will show you your search analytics (and let you request indexing), you must prove you own the domain. This is exactly the flow in the two screenshots — recreated here so you can talk through it step by step.

Step 1 — Pick a property type

Recommended
🌐

Domain

  • All URLs across every subdomain (m., www., …)
  • Both https and http
  • Requires DNS verification
{{ domain() }}
or
🔗

URL prefix

  • Only URLs under the exact address you enter
  • Only the protocol you specify
  • Allows multiple verification methods (HTML file, tag…)
https://www.{{ domain() }}

Step 2 — Google hands you a DNS TXT record

Verify domain ownership via DNS record {{ domain() }}
  1. 1 Record type: TXT (recommended)
  2. 2 Sign in to your domain provider (e.g. Namecheap, GoDaddy)
  3. 3 Copy this record into the DNS configuration for {{ domain() }}:
    {{ verificationRecord() }}
  4. 4 Press Verify. (DNS can take minutes to hours to propagate.)

Step 3 — Paste it at your registrar (Namecheap)

In Namecheap: Domain List → Manage → Advanced DNS → Add New Record. You’re telling the world’s DNS “this random Google string lives on my domain” — only the real owner could do that, which is how Google trusts you.

TypeHostValueTTL
TXT Record @ {{ verificationRecord() }} Automatic

@ means the root domain ({{ domain() }} itself). Save, wait, then click Verify back in Search Console.

✅
Once verified you can submit your sitemap.xml, watch which queries bring traffic, and use URL Inspection → Request indexing to nudge Google to re-crawl a page after you ship new SEO tags or SSR. The verification TXT must stay in DNS — remove it and Google eventually drops your ownership.
